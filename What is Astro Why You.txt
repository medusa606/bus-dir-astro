What is Astro & Why You Should 100% Use It
Astro is a static site generator — it runs at build time, fetches all your listings from Supabase, and outputs real .html files for every single page. No JavaScript framework required on the client.

Why it's perfect for your directory:
Requirement	Astro's Answer
Individual listing pages	Generates a real HTML file for each listing
Hierarchical SEO URLs	File-based dynamic routing: /UK/Bristol/Easton/ZamZam
Fast page loads	Ships zero JS by default — just HTML + CSS
SEO	Search engines get fully-rendered HTML, not a loading spinner
Free hosting	Deploys perfectly to Cloudflare Pages
Your existing CSS	Paste it straight in — Astro uses standard HTML/CSS
Your current setup (single index.html fetching from Supabase client-side) means Google sees "Loading local businesses..." — that's terrible for SEO. Astro fixes this completely.

Step 1: Update Your Supabase Table
Add these columns to your listings table for the hierarchical URL structure:

sql
-- Run in Supabase SQL Editor
ALTER TABLE listings
ADD COLUMN IF NOT EXISTS country_slug TEXT DEFAULT 'uk',
ADD COLUMN IF NOT EXISTS city_slug TEXT,
ADD COLUMN IF NOT EXISTS area_slug TEXT,
ADD COLUMN IF NOT EXISTS business_slug TEXT;

-- Create a unique index for the full URL path
CREATE UNIQUE INDEX idx_listing_full_slug 
ON listings(country_slug, city_slug, area_slug, business_slug);

-- Helper function to auto-generate slugs from names
CREATE OR REPLACE FUNCTION generate_slug(input TEXT)
RETURNS TEXT AS 
$$
BEGIN
  RETURN lower(regexp_replace(regexp_replace(trim(input), '[^a-zA-Z0-9\s-]', '', 'g'), '\s+', '-', 'g'));
END;
$$
 LANGUAGE plpgsql;
Then update your existing rows:

sql
-- Example: Update a listing
UPDATE listings SET
  country_slug = 'uk',
  city_slug = 'bristol',
  area_slug = 'easton',
  business_slug = 'zamzam'
WHERE name = 'ZamZam';

export async function getStaticPaths() {
    const { data: listings } = await supabase
        .from('listings')
        .select('*')

    return listings.map(listing => ({
        params: {
            country: listing.country_slug,
            city: listing.city_slug,
            area: listing.area_slug,
            business: listing.business_slug,

export async function getStaticPaths() {
    const { data: listings } = await supabase
        .from('listings')
        .select('country_slug, city_slug, area_slug')
        .eq('status', 'Published');

    // Get unique area combinations
    const unique = [...new Set(listings.map(l => 
        JSON.stringify({ country: l.country_slug, city: l.city_slug, area: l.area_slug })
    ))].map(s => JSON.parse(s));

    return unique.map(u => ({
        params: { country: u.country, city: u.city, area: u.area }
    }));
}

Step 7: Auto-Rebuild When Data Changes
Since Astro generates static pages at build time, you need to trigger a rebuild when you add/edit listings in Supabase. Two approaches:

Option A: Cloudflare Deploy Hook (Easiest)
In Cloudflare Pages → Settings → Builds → Deploy hooks
Create a hook — you get a URL like https://api.cloudflare.com/client/v4/pages/webhooks/deploy_hooks/abc123
In Supabase, create a Database Webhook or Edge Function that hits this URL when the listings table changes
Option B: Manual rebuild
Just push any commit to GitHub, or click "Retry deployment" in Cloudflare.
